**Модуль мониторинга (task1_monitoring.py)**

Класс PopulationMonitor загружает исторические данные из CSV файлов и предоставляет следующие возможности:

Загрузка данных - чтение CSV файлов с историческими данными о населении городов за 2015-2024 годы

Фильтрация городов - исключение поселков городского типа и малых населенных пунктов, работа только с городами от 50 000 жителей

Классификация городов - распределение по категориям: город-миллионник, крупный город, средний город, большой город, малый город

Расчет динамики - вычисление абсолютного и относительного изменения населения, среднегодового темпа роста (CAGR)

Демографические показатели - расчет коэффициентов рождаемости и смертности на основе трендов населения

Региональная агрегация - суммирование населения по регионам для тепловых карт

Топ изменений - выявление городов с наибольшим ростом и снижением населения

**Формула CAGR (Compound Annual Growth Rate):**

CAGR = (End_Population / Start_Population) ^ (1 / Years) - 1

Модуль прогнозирования (task2_forecasting.py)

Класс PopulationForecaster использует машинное обучение для прогнозирования численности населения:

Линейная регрессия - модель scikit-learn LinearRegression обучается на исторических данных (год -> население)

Прогноз на горизонт - генерация прогнозов на 5, 10 или 15 лет вперед

Доверительные интервалы - расчет 95% доверительного интервала на основе стандартного отклонения остатков (z-score = 1.96)

**Метрики качества - оценка точности модели:**

MAPE (Mean Absolute Percentage Error) - средняя абсолютная процентная ошибка

RMSE (Root Mean Square Error) - среднеквадратичная ошибка

MAE (Mean Absolute Error) - средняя абсолютная ошибка

R-squared - коэффициент детерминации

Сценарное прогнозирование - три сценария развития:

Оптимистичный (+2% в год)

Базовый (исторический тренд)

Пессимистичный (-1% в год)

Кросс-валидация - проверка модели на исторических данных для расчета метрик

**Как устроена база данных**

**Тип базы данных**

SQLite 3 - встраиваемая реляционная база данных без необходимости установки сервера. Файл базы данных forecasts.db создается автоматически в корневой директории проекта.

**Схема базы данных**

CREATE TABLE saved_forecasts (

    id INTEGER PRIMARY KEY AUTOINCREMENT,
    
    city TEXT NOT NULL,
    
    horizon INTEGER NOT NULL,
    
    forecast_data TEXT NOT NULL,
    
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
    
);

CREATE INDEX idx_city ON saved_forecasts(city);

**Описание полей:**

id - автоинкрементный первичный ключ

city - название города (индексированное поле для быстрого поиска)

horizon - горизонт прогноза в годах (5, 10, 15)

forecast_data - полные данные прогноза в формате JSON (включает исторические данные, прогноз, доверительные интервалы, метрики)

created_at - временная метка создания записи

updated_at - временная метка последнего обновления

**Особенности реализации**

WAL режим (Write-Ahead Logging) - улучшает производительность при одновременных чтении и записи

Автоматическая инициализация - таблица создается при первом импорте модуля

Защита от повреждения - автоматическое восстановление при обнаружении ошибок структуры

Дедупликация - при сохранении проверяется наличие записи для того же города с тем же горизонтом, существующая запись обновляется

JSON сериализация - данные прогноза преобразуются в JSON строку с обработкой numpy типов
